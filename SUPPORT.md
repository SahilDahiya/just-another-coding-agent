# Support

## Where to start

- Read [README.md](README.md) for install and launch paths.
- Read [TROUBLESHOOTING.md](TROUBLESHOOTING.md) for known failure modes and bug
  report guidance.
- Read [docs/README.md](docs/README.md) for architecture and contract docs.

## If you think you found a bug

Open a GitHub issue and include:

- JACA version
- OS and environment
- exact error text
- reproduction steps
- any relevant logs or stdout/stderr captures

The troubleshooting guide already lists the highest-signal details to include.

## If you want to contribute

Read [CONTRIBUTING.md](CONTRIBUTING.md) first. JACA has strong architectural
constraints, so proposed changes are much easier to review when they are framed
against the existing contract and design docs.

## Security issues

Do not report suspected vulnerabilities in public issues first. See
[SECURITY.md](SECURITY.md).
